3、ScaleTransition实现缩放动画

  • 首先 Flutter 中的动画分为两类：1.补间动画（Tween）2.基于物理的动画

  • Flutter 中的动画系统基于 「Animation」，「Widgets」 可以直接将这些动画合并到自己的 build 方法中来读取它们的当前值或者监听它们的状态变化，或者可以将其作为的更复杂动画的基础传递给其他 widgets。

  • 「Animation」 的状态有如下几种：

    1.dismissed：一般情况，动画会从这个状态开始

    2.forward：运行时可能是这个

    3.reverse：：运行时也可能是这个

    4.completed：完成的时候会变成这个

  • Hero动画就是在路由切换时，有一个共享的Widget可以在新旧路由间切换，由于共享的Widget在新旧路由页面上的位置、外观可能有所差异，所以在路由切换时会逐渐过渡，这样就会产生一个Hero动画。

  • 要触发Hero动画，Hero必须存在于新页面动画的第一帧。

  • 并且一个路由里只能有一个Hero 的 tag。

28、flutter 报错 DioError [DioErrorType.DEFAULT]: Bad state: Insecure HTTP is not allowed by platform

network_security_config.xml

29、在Flutter中路由分为静态路由跟动态路由两种：

静态路由指的是需要提前把各个需要跳转的页面路径注册在routes: <String, WidgetBuilder> {}中，且静态路由不支持向下一个页面传递参数，但是可以接收下一个页面的返回值。（新版本flutter已经支持静态路由传参）
动态路由使用就相对来说比较灵活一点，动态路由同样支持向下一个页面传递参数，而且在使用时不需要我们提前规划好页面路径，只需要在具体跳页逻辑中自己去构造MaterialPageRoute对象来完成页面跳转，或者用PageRouterBuilder来自定义路由跳转时的动画，

1、未初始化变量声明 var/ dynamic/ object

声明初始化变量

  • 如果没有初始值，var可以变成任意类型；

  • dynamic动态任意类型，编译阶段不检查类型；

  • object动态任意类型，编译阶段检查类型。

  • 如果用var声明一个变量，如果已经指定类型，之后不可以改变；但object可以改变。

  • dynamic在编译时不会检查类型，不会报错；但object在编译时会检查类型，报红并提示没有方法。

计算机网络一些基础知识的总结
方便以后复习

网络分层模型

1、 物理层：在数据链路实体之间，合理的通过中间系统，为位传输所需要的物理连接的激活，保持和取消提供机械的，规程的，电气的，功能的手段。
负责产生 0、1 比特流；
2、 数据链路层：基本功能是将不可靠的物理链路变为可靠的数据链路，为网络层提供可靠的数据传输。组装数据帧，透明传输，差错检测。
负责将比特流封装成帧，帧中包含源 MAC 地址和 目标 MAC 地址
3、网络层：主要功能有存储转发，路由选择，拥塞控制和网络互联等。
IP 协议是 TCP/IP 协议的核心，所有的 TCP、UDP、ICMP 都以 IP 数据格式传输。
4、 传输层：实现差错控制和拥塞控制等，实现端到端的，可靠或不可靠的数据传输服务。
5、会话层：管理主机之间的会话进程，负责建立，管理，终止进程之间的会话。会话层还利用在数据中插入校验点来实现数据的同步。
6、表示层：对上层传输的信息进行变换，保证一个主机的应用层信息可以被另一个主机的应用程序所理解。表示层的数据转换包括加密，压缩，格式转换等。
7、应用层：用户的应用程序提供网络服务的接口。将用户的操作通过应用程序转换成为服务，并匹配一个相应的服务协议发送给传输层。
